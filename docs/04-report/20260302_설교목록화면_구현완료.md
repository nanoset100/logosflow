## 2. 구현 완료 항목

### 2.1 데이터 모델 (SermonModel)

**파일**: `lib/data/models/sermon_model.dart`

#### 구현 내용
```
필드:
  - id: 설교 고유 ID
  - churchCode: 교회 코드
  - title: 설교 제목
  - date: 설교 날짜
  - pastor: 담당 목사명
  - bibleVerse: 성경 본문
  - summary: 설교 요약
  - audioUrl: 음성 파일 URL (선택)
  - devotionals: 주중 묵상 (day1~day5)
  - keyPoints: 핵심 포인트 리스트
  - createdAt: 생성 일시
```

#### 주요 기능
- **fromFirestore()**: Firestore 문서 → 모델 변환
- **toFirestore()**: 모델 → Firestore 직렬화
- **formattedDate**: 한글 형식 날짜 표시 (예: "2026년 2월 23일")
- **dayOfWeek**: 요일 표시 (예: "월요일")

#### 코드 라인 수
- 총 76줄 (모델 정의 + 직렬화 + 편의 메서드)

### 2.2 설교 서비스 (SermonService)

**파일**: `lib/data/services/sermon_service.dart`

#### 구현 메서드

| 메서드 | 반환값 | 설명 |
|--------|--------|------|
| `getSermons(churchCode)` | `Stream<List<SermonModel>>` | 교회의 설교 목록 실시간 조회 (최신순) |
| `getSermon(churchCode, sermonId)` | `Future<SermonModel?>` | 단일 설교 조회 |
| `getLatestSermon(churchCode)` | `Future<SermonModel?>` | 최신 설교 1개 조회 |
| `addSermon(sermon)` | `Future<String?>` | 설교 추가 (목사용) |
| `updateSermon(churchCode, sermonId, updates)` | `Future<bool>` | 설교 수정 (목사용) |
| `deleteSermon(churchCode, sermonId)` | `Future<bool>` | 설교 삭제 (목사용) |

#### 특징
- Firestore의 스트림 기반 실시간 데이터 연동
- 에러 핸들링 구현 (try-catch)
- 목사용 CRUD 메서드 구조 완성 (권한 검증은 Firestore 규칙에서 처리)

#### 코드 라인 수
- 총 112줄 (6개 메서드 + 에러 처리)

### 2.3 설교 목록 화면 (SermonListScreen)

**파일**: `lib/presentation/screens/sermon/sermon_list_screen.dart`

#### 주요 기능
1. **SharedPreferences 연동**
   - 로그인 시 저장된 churchCode 로드
   - 교회 코드 기반 데이터 필터링

2. **StreamBuilder 기반 UI**
   - Firestore 실시간 연결
   - 로딩, 에러, 빈 상태 처리

3. **_SermonCard 위젯**
   - 날짜 + 요일 표시 (파란색 배지)
   - 설교 제목 (18pt 굵게)
   - 성경 본문 (아이콘 포함)
   - 설교 요약 (2줄 말림표시)
   - 담당 목사명 (회색 표시)

4. **빈 상태 UI**
   - 책 아이콘 표시
   - "아직 설교가 없습니다" 안내 문구

5. **네비게이션**
   - 카드 탭 → SermonDetailScreen 이동
   - Material 3 디자인 준수

#### 화면 구성
```
┌─────────────────────────┐
│ 설교 노트  [로그아웃]      │  ← AppBar
├─────────────────────────┤

│  │ 우리는 그리스도   │  │  ← _SermonCard

│  │ 담임목사님       │  │

└─────────────────────────┘
```

#### 코드 라인 수
- 총 221줄 (상태 관리 + UI 레이아웃 + 카드 위젯)

## 7. 발견된 문제 및 해결

### 문제 1: 날짜 포맷팅
**상황**: Firestore Timestamp를 한글 형식으로 표시 필요
**해결**: `formattedDate` getter와 `dayOfWeek` getter 구현
**결과**: "2026년 2월 23일 (월요일)" 정상 표시

### 문제 2: churchCode 로드
**상황**: 로그인 후 저장된 교회 코드를 SharedPreferences에서 로드
**해결**: `AuthService.getSavedChurchCode()` 메서드 활용
**결과**: StreamBuilder 초기화 전 churchCode 로드 완료

### 문제 3: 빈 상태 처리
**상황**: 설교 데이터가 없을 때 사용자 경험 개선
**해결**: 아이콘 + 안내 문구가 포함된 빈 상태 UI 구현
**결과**: 더 나은 UX 제공

### A1. SermonModel fromFirestore 메서드
```dart
factory SermonModel.fromFirestore(DocumentSnapshot doc) {
  final data = doc.data() as Map<String, dynamic>;

  return SermonModel(
    id: doc.id,
    churchCode: data['churchCode'] ?? '',
    title: data['title'] ?? '',
    date: (data['date'] as Timestamp).toDate(),
    pastor: data['pastor'] ?? '',
    bibleVerse: data['bibleVerse'] ?? '',
    summary: data['summary'] ?? '',
    audioUrl: data['audioUrl'],
    devotionals: Map<String, String>.from(data['devotionals'] ?? {}),
    keyPoints: List<String>.from(data['keyPoints'] ?? []),
    createdAt: (data['createdAt'] as Timestamp).toDate(),
  );
}
```

### A2. SermonService getSermons 메서드
```dart
Stream<List<SermonModel>> getSermons(String churchCode) {
  return _firestore
      .collection('churches')
      .doc(churchCode)
      .collection('sermons')
      .orderBy('date', descending: true)
      .snapshots()
      .map((snapshot) {
    return snapshot.docs
        .map((doc) => SermonModel.fromFirestore(doc))
        .toList();
  });
}
```

### A3. SermonListScreen StreamBuilder
```dart
StreamBuilder<List<SermonModel>>(
  stream: _sermonService.getSermons(_churchCode!),
  builder: (context, snapshot) {
    if (snapshot.connectionState == ConnectionState.waiting) {
      return const Center(child: CircularProgressIndicator());
    }

    final sermons = snapshot.data ?? [];

    return ListView.builder(
      padding: const EdgeInsets.all(16),
      itemCount: sermons.length,
      itemBuilder: (context, index) {
        return _SermonCard(
          sermon: sermons[index],
          onTap: () => Navigator.push(
            context,
            MaterialPageRoute(
              builder: (context) => SermonDetailScreen(
                sermon: sermons[index],
              ),
            ),
          ),
        );
      },
    );
  },
)
```
