# 말씀브릿지(WordBridge): 플랫폼 설계 및 자동화 파이프라인

본 문서는 대형교회 9곳을 타겟으로 하는 '선제작 후협상' 전략 기반의 말씀브릿지 플랫폼 개발을 위한 핵심 요소인 **1) 홈 화면 프로토타입 UI**와 **2) AI 자동화 데이터 파이프라인**을 상세히 설계한 내용입니다.

---

## 1. 홈 화면 프로토타입 UI 시스템 설계

말씀브릿지의 첫 홈 화면은 성도들의 선택 피로도를 낮추고 신뢰감을 주기 위해 심플하고 직관적이어야 합니다. 전체 UI 모티브는 대중적으로 친숙한 '토스(Toss)'나 기존 '대표기도' 앱의 레이아웃을 참조합니다.

### 1.2 핵심 UI/UX 포인트
*   **헤드라인 강조**: 성도들의 시선이 가장 먼저 머무는 곳은 **'목사님 이름'**입니다. 글씨 크기를 가장 크게 하고 폰트 두께를 볼드(Bold)로 처리합니다.
*   **서브 타이틀**: 소속 교회명은 작고 연한 회색(Grey)으로 처리하여 컴플레인을 최소화하면서도 소속을 밝혀 신뢰도를 줍니다.
*   **비주얼 에셋(아이콘)**: 교회 로고 상표권 분쟁을 막기 위해 **목사님의 1D/2D 캐리커처 일러스트**를 사용합니다. (AI 이미지 생성기로 쉽게 제작 가능)
*   **인바운드(영업) 버튼**: 하단에 강력한 Call To Action (CTA) 버튼인 "**우리 교회 추가 신청하기**"를 배치하여 구글 폼(Google Forms)으로 연결합니다. 중소형 교회 성도들이 자기 교회 목사님 설교도 요약해 달라고 요청하게 만듭니다. (이 DB가 추후 B2B 영업의 핵심 자산이 됩니다!)

## 2. AI 데이터 자동화 파이프라인 (Data Pipeline) 설계

매주 월요일 아침 6시 알림을 보내려면 주일 오후~밤 사이에 9개 교회의 설교 영상에서 오디오를 추출하고, 요약하여 DB에 넣는 작업이 완료되어야 합니다. 사람이 수동으로 하면 금방 지치기 때문에 **100% 자동화된 백엔드 시스템**이 구축되어야 합니다.

### 2.1 아키텍처 개요 (클라우드 + AI 연동)

이 자동화 봇은 AWS Lambda 또는 구글 Cloud Functions나 외부 서버에서 타이머(Cron)에 의해 매주 일요일 저녁 6시에 자동 실행됩니다.

```mermaid
graph TD
    A[Cron Job: 매주 일요일 18:00 실행] --> B[유튜브 채널 크롤링]
    B -->|최신 주일설교 감지| C{설교 영상 발견?}
    C -->|Yes| D[오디오(MP3) 추출 다운로드]
    C -->|No| E[1시간 후 재시도]
    
    D --> F[Whisper AI (STT API)]
    F -->|오디오 → 전체 텍스트 변환| G[OpenAI GPT-4o API]
    
    G -->|프롬프트: 5일치 묵상 생성| H[JSON 구조화 데이터]
    H --> I[Firebase Firestore DB 저장]
    I --> J[FCM: 월요일 아침 푸시 알림 예약]
```

### 2.2 각 단계별 세부 구현 기술

#### 단계 1: 유튜브 자동 크롤링 (Python)
*   **사용 도구:** `yt-dlp` 라이브러리 (또는 YouTube Data API)
*   **방식:** 9개 대형교회의 공식 유튜브 채널 중 '주일설교' 플레이리스트를 주시합니다. 일요일 날짜(예: 2026-03-08)가 포함된 최신 영상이 올라오면 URL을 수집합니다.
*   **동작:** 영상 전체를 다운받지 않고, 서버 리소스 절약을 위해 **오디오(mp3)만 추출**하여 임시 서버에 다운로드 받습니다.

#### 단계 2: 음성 텍스트 변환 (STT - Speech to Text)
*   **사용 도구:** OpenAI `Whisper API` (또는 로컬 서버 구동)
*   **방식:** 다운로드 받은 mp3 파일을 Whisper API 모델에 전송하여 전체 설교의 자막 텍스트 원문판(Transcript)을 생성합니다. (약 30분짜리 설교 기준 API 비용 매우 저렴함)

#### 단계 3: AI 요약 및 5일 묵상 생성 (Prompt Engineering)
*   **사용 도구:** OpenAI `GPT-4o` (또는 Claude 3.5 Sonnet)
*   **방식:** 변환된 STT 원문을 LLM(대형 언어 모델)에 던지면서 강력한 '시스템 프롬프트'를 적용합니다.
*   **핵심 프롬프트 예시:**
    ```text
    너는 탁월한 신학자이자 목회 코치야. 
    다음 제공되는 [설교 원문]을 분석해서, 평신도들이 월~금요일까지 5일 동안 매일 5분씩 묵상할 수 있는 콘텐츠를 작성해 줘.
    결과는 반드시 아래 JSON 형식으로 뱉어내야 해.
    {
      "title": "설교 제목 요약 (15자 이내)",
      "bible_verse": "본문 성경 구절 (예: 로마서 8:1-2)",
      "day1": "월요일: 핵심 요약 (설교의 가장 중요한 메시지 재확인)",
      "day2": "화요일: 질문과 성찰 (설교 내용을 내 삶에 비추어 보는 질문)",
      ...
      "day5": "금요일: 삶의 적용과 결단 (이번 주말을 앞두고 실천할 작은 행동)"
    }
    ```

#### 단계 4: 데이터베이스 자동 적재
*   **사용 도구:** Firebase Admin SDK
*   **방식:** GPT-4가 뱉어낸 완벽한 JSON 포맷 데이터를 우리 앱의 서버인 `Firebase Cloud Firestore`에 해당 목사님의 고유 ID 데이터(Document)로 자동 Update 합니다.

#### 단계 5: 푸시 알림 (Push Notification)
*   **사용 도구:** Firebase Cloud Messaging (FCM)
*   **방식:** 데이터가 성공적으로 올라가면, 다음 날 월요일 아침 7시(사용자 설정 시간)에 해당 목사님을 구독한 유저들에게 푸시 알림이 발송되도록 타겟팅 메시지를 예약합니다.
    *   *메시지:* "알림: 이찬수 목사님의 이번 주 새로운 말씀 묵상이 도착했습니다! 🙌"

## 3. 요약 및 권장 액션

이 시스템을 구축하면 사람이 일일이 타이핑하거나 설교를 듣고 요약할 필요 없이, 파이프라인이 **매주 수십 명의 유명 목사님들 설교를 알아서 다이제스트(Digest)**하여 앱에 공급하는 강력한 "**콘텐츠 자동 생산 공장**"이 만들어집니다.

1.  침신 말씀노트를 클론하여 새로운 프로젝트 (WordBridge) 생성.
2.  위의 홈 화면 UI(9개 버튼)를 플러터(Flutter)로 구현.
3.  파이썬(Python) 기반의 유튜브 크롤링 + 자동 요약 스크립트 작성 테스트.
4.  테스트로 1~2명의 목사님 설교 URL을 넣어 파이프라인이 정상 작동하는지 POC(개념 증명) 진행.
