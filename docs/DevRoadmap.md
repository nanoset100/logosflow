## Phase 0: 준비 (2026년 3월)

### 목표
- 개발 환경 완전 셋업
- Firebase 프로젝트 구성
- 기술 스택 검증
- 파일럿 교회 확보

### 주차별 계획

#### 1주차 (3/1 ~ 3/7): 환경 셋업
```
개발 환경:
□ Flutter SDK 3.16+ 설치
□ Dart SDK 확인
□ VS Code / Android Studio 설정
□ Flutter 플러그인 설치

Firebase 프로젝트:
□ Firebase 프로젝트 생성 (dev/staging/prod)
□ iOS/Android 앱 등록
□ google-services.json / GoogleService-Info.plist
□ Firebase Authentication 활성화
  - 이메일/비밀번호
  - Google 로그인
□ Firestore Database 생성
□ Firebase Storage 생성
□ Firebase Cloud Messaging 설정

버전 관리:
□ GitHub 레포지토리 생성
□ 브랜치 전략 수립 (main/develop/feature)
□ .gitignore 설정 (API 키 제외)
□ 팀 접근 권한 설정
```

#### 2주차 (3/8 ~ 3/14): 기반 코드 구축
```
Flutter 프로젝트:
□ Flutter 프로젝트 초기화
□ pubspec.yaml 의존성 설정
□ 프로젝트 폴더 구조 생성
□ 테마/색상/타이포그래피 설정
□ GoRouter 기본 설정

Firebase 연동:
□ Firebase Core 초기화
□ Firestore Security Rules 작성 (초안)
□ Cloud Functions 환경 설정
□ 로컬 Firebase Emulator 설정

카카오 SDK:
□ 카카오 개발자 계정 생성
□ 애플리케이션 등록
□ 카카오 로그인 SDK 통합 테스트
```

#### 3주차 (3/15 ~ 3/21): AI 연동 테스트
```
Pastors.ai API:
□ Pastors.ai 계정 생성 (Team 플랜 $75/월)
□ API 키 발급
□ 설교 샘플 업로드 테스트
□ 결과 품질 검토 (한국어 지원 확인)
□ API 응답 시간 측정

대안 준비 (GPT-4):
□ OpenAI API 계정 생성
□ 설교 요약 프롬프트 작성
□ Whisper STT 테스트 (한국어)
□ 비용 계산 (설교당 $0.5 검증)

결정:
□ Pastors.ai 품질이 충분하면 Phase 1 사용
□ 품질 미달 시 즉시 GPT-4 전환
```

#### 4주차 (3/22 ~ 3/31): 파일럿 교회 확보
```
영업/파트너십:
□ 파일럿 대상 교회 리스트 작성 (10개)
□ 담임목사 컨택 (지인 네트워크)
□ 데모 시연 미팅 (3개 교회 이상)
□ 파일럿 협약 체결 (3개 교회 목표)

파일럿 교회 조건:
- 교인 50명 이상
- 담임목사 직접 참여 의사
- 앱 서비스 2개월 이상 사용 의지
- 피드백 제공 동의

계약 내용:
- 파일럿 기간: 3개월 무료
- 설교 수동 처리 우리가 제공
- 주 1회 현황 미팅
- 데이터 활용 동의
```

### 주요 리스크 및 대응

| 리스크 | 발생 가능성 | 영향도 | 대응 방안 | 담당 |
|--------|------------|-------|----------|------|
| AI 품질 미달 | 중간 | 높음 | 인간 편집 + 목사 검수 | PM |
| 앱스토어 거절 | 낮음 | 높음 | 가이드라인 사전 검토 | 개발 |
| 파일럿 이탈 | 중간 | 높음 | 전담 CS, 즉시 지원 | PM |
| AI 비용 폭증 | 낮음 | 중간 | 비용 한도 설정, 모니터링 | 개발 |
| 개인정보 유출 | 매우 낮음 | 매우 높음 | Security Rules, 암호화 | 개발 |
| 총회 추천 실패 | 낮음 | 높음 | 침신대 내부 지원 확보 | PM |

### 주간 리스크 리뷰

```
매주 월요일 30분:
- 지난 주 이슈 점검
- 이번 주 리스크 레벨 재평가
- 필요 시 대응 방안 조정
- 블로커 즉시 에스컬레이션
```

## 커뮤니케이션 계획

### 이해관계자별 업데이트

```
침신대 파트너:
- 월 1회 정기 보고 (첫째 주 화요일)
- 파일럿 현황 공유
- 주요 결정 사항 협의

파일럿 교회:
- 주 1회 담임목사 미팅
- 실시간 앱 현황 대시보드
- 긴급 이슈 24시간 대응

개발팀 내부:
- 일일 스탠드업 (10분)
- 주간 스프린트 리뷰
- 격주 회고 미팅
```

### 결정 로그

```
모든 주요 기술/비즈니스 결정을
docs/decisions/ 폴더에 ADR(Architecture Decision Record) 형태로 기록

예시:
- ADR-001: Flutter vs React Native 선택
- ADR-002: Pastors.ai vs GPT-4 선택
- ADR-003: Firebase vs Supabase 선택
```
