## 1. 제품 개요

### 1.1 제품명
- **국문**: 말씀브릿지
- **영문**: WordBridge
- **부제**: AI 말씀비서가 설교를 삶으로 연결합니다

### 1.2 핵심 가치 제안
> "어떤 교회든, 어떤 교단이든 - 설교가 삶이 되게 합니다"

말씀브릿지는 특정 교단에 종속되지 않는 **범용 AI 말씀비서**입니다. 전국 모든 교회에서 사용 가능하며, 교회의 규모와 교단에 관계없이 AI를 통해 설교를 삶으로 연결하는 서비스입니다.

### 1.3 브랜드 포지셔닝

```
          교회 특화성
               ↑
    침신 말씀노트
    (침례교 전용)    ●
                         말씀브릿지
                               ●
    ←─────────────────────────────→
    소규모                   대규모
    교회                     교회
                    갓피플 ●
                         (개인용)
               ↓
          개인 중심
```

**말씀브릿지 포지션**: 중-대형 교회, 모든 교단, 교회 단위 서비스

### 1.4 침신 말씀노트 vs 말씀브릿지 비교

| 항목 | 침신 말씀노트 | 말씀브릿지 |
|------|-------------|-----------|
| 대상 | 침례교 전용 | 모든 교회 |
| 브랜딩 | 침신대 공식 | 독립 브랜드 |
| 가격 | 교인당 250원 | 교인당 300원 |
| 교단 특화 | 침례교 교단 연동 | 교단 중립 |
| 콘텐츠 | 침례교 신학 기반 | 범교단적 |
| 네트워크 | 침신대 채널 | 독립 영업 |

## 2. 시장 분석

### 2.1 시장 규모

```
국내 교회 현황:
- 총 교회 수: 약 60,000개
- 개신교 등록 성도: 약 960만 명
- 주요 교단:
  장로교 (통합/합동): 35%
  감리교: 13%
  침례교: 6%
  성결교: 5%
  기타: 41%

타겟 시장:
- 교인 50명 이상 교회: 약 15,000개
- 교인 100명 이상 교회: 약 8,000개
- 교인 300명 이상 교회: 약 2,000개

TAM (전체 시장):
60,000개 × 150명 × 300원 = 27억원/월 = 324억원/년

SAM (실제 공략 가능):
8,000개 × 150명 × 300원 = 3.6억원/월 = 43억원/년

SOM (3년 내 목표):
500개 × 150명 × 300원 = 2,250만원/월 = 2.7억원/년
```

### 2.2 경쟁 분석

| 경쟁사 | 유형 | 강점 | 약점 | 대응 전략 |
|--------|------|------|------|----------|
| 갓피플 | 성경/큐티 앱 | 브랜드 인지도 | 교회 연동 없음 | 교회 맞춤형 강조 |
| 처치앱 | 교회 관리 앱 | 공지/출결 기능 | 신앙 성장 없음 | AI 기능 차별화 |
| 유튜브 교회 채널 | 영상 플랫폼 | 무료 | AI 요약 없음 | 자동화 강조 |
| 설교닷컴 | 설교 아카이브 | 많은 설교 | 개인화 없음 | 맞춤형 적용 |
| 카카오채널 | 메시지 앱 | 친숙함 | 신앙 기능 없음 | 전문성 강조 |

### 2.3 차별화 요소

```
말씀브릿지만의 3가지 차별점:

1. 📊 AI 설교 → 삶 연결
   - 우리 교회 설교를 AI가 5일로 변환
   - 개인화된 적용 가이드
   - 삶의 맥락에 맞는 실천법

2. 🤝 목사-성도 연결
   - 목사님이 성도의 참여 현황 확인
   - 격려 필요 성도 자동 알림
   - 목회 인사이트 대시보드

3. 🌱 따뜻한 AI 격려
   - 개인 상황별 맞춤 격려
   - 포기하지 않게 도와주는 AI
   - 종교적이지만 부담 없는 톤
```

## 3. 타겟 사용자 세분화

### 3.1 교회 규모별 세분화

**Tier 1: 중형 교회 (100-500명)**
```
특징:
- 목사 1-3인 체제
- IT 인프라 기본 수준
- 예산 제한적이나 혁신 의지 있음
- 앱 도입 결정권: 담임목사

Pain Point:
- 성도 개별 관리 어려움
- 주중 신앙 활동 프로그램 부족
- AI/디지털 경쟁력 부족 인식

우리 접근:
- 목사 1:1 데모
- 무료 파일럿 제공
- ROI 명확히 제시
```

**Tier 2: 대형 교회 (500-3000명)**
```
특징:
- 부서별 목사, 전문 스태프
- IT 시스템 이미 있음
- 자체 앱 보유 교회도 있음
- 결정 프로세스 복잡

Pain Point:
- 소그룹 관리 복잡
- 개인화 목양 한계
- 청장년 이탈 방지 필요

우리 접근:
- 소그룹 리더 통한 보급
- 기존 시스템과 통합 옵션
- B2B 맞춤 제안
```

**Tier 3: 소형 교회 (50-100명)**
```
특징:
- 목사 단독 운영
- 예산 매우 제한
- IT 역량 낮음
- 성장 의지 강함

Pain Point:
- 비용 부담
- 사용 방법 학습 어려움
- 관리 인력 없음

우리 접근:
- 저렴한 요금제
- 자동화로 관리 부담 최소
- 동영상 튜토리얼 제공
```

### 3.2 교단별 세분화

```
장로교 (통합/합동):
- 특징: 보수적, 장로 의사결정
- 전략: 장로회 추천 채널
- 콘텐츠: 개혁주의 신학 기반

감리교:
- 특징: 사회적 실천 강조
- 전략: 감리교 총회 채널
- 콘텐츠: 사회 참여 적용 강조

성결교:
- 특징: 성령 체험 강조
- 전략: 개별 교회 영업
- 콘텐츠: 영적 은사 적용 포함

기독교대한교회 (기장):
- 특징: 진보적, 에큐메니컬
- 전략: 신학대학 통한 접근
- 콘텐츠: 사회 정의 실천 강조
```

## 4. 기능 요구사항

### 4.1 범용 기능 (공통 기반)

침신 말씀노트와 동일한 핵심 기능 + 범용성 강화:

#### 4.1.1 다교단 지원

```
교단 설정:
- 장로교(통합), 장로교(합동), 감리교,
  침례교, 성결교, 순복음, 기타 직접 입력

교단별 조정:
- 예배 형식 (정기 예배 일정)
- 신학적 강조점 (콘텐츠 톤 조정)
- 절기/교회 달력 (교단 달력 지원)

중립적 기본 설정:
- 특정 교단 편향 없는 기본 콘텐츠
- 목사가 신학적 방향 커스터마이징 가능
```

### 4.3 AI 기능 고도화

#### 4.3.1 GPT-4 기반 콘텐츠 생성 (Phase 2)

```python
# 말씀브릿지 AI 콘텐츠 생성 파이프라인

class WordBridgeAI:
    """
    Phase 2부터 자체 AI 시스템 구축
    """

    def process_sermon(self, audio_url: str, church_config: dict) -> SermonContent:
        """
        1단계: Whisper로 음성 → 텍스트
        2단계: GPT-4로 콘텐츠 생성
        3단계: 교단/교회 톤 맞춤 조정
        4단계: 신학자 검수 (자동 플래그)
        """
        transcript = self.whisper_stt(audio_url)
        content = self.gpt4_generate(
            transcript=transcript,
            denomination=church_config['denomination'],
            tone=church_config['tone_preference'],
            pastor_style=church_config['pastor_style']
        )
        return self.quality_check(content)

    def generate_daily_devotion(
        self,
        sermon: SermonContent,
        member: Member,
        day: int  # 1-5
    ) -> DailyContent:
        """
        개인화된 일일 묵상 생성
        - 나이, 직업, 이전 묵상 패턴 반영
        - 어른 친화적 언어 자동 조정
        """
        context = {
            'sermon': sermon,
            'member_age': member.age_group,
            'day_theme': DAY_THEMES[day],
            'previous_notes': member.get_recent_notes(3)
        }
        return self.generate_personalized_content(context)
```

## 6. 온보딩 프로세스

### 6.1 교회 온보딩 (B2B)

```
Step 1: 교회 등록 (관리자)
→ 사업자등록번호 또는 교회 인증
→ 목사 이메일 인증
→ 교회 기본 정보 입력

Step 2: 교회 설정
→ 교단 선택
→ 예배 일정 설정
→ 교회 로고/색상
→ 교인 초대 방법 선택

Step 3: 첫 설교 업로드 (목사)
→ 업로드 방법 선택
→ AI 처리 대기
→ 결과 검토 및 승인

Step 4: 교인 초대
→ 교회 코드 공유 (카카오, 문자)
→ 주보/광고 템플릿 제공
→ QR 코드 생성

Step 5: 파일럿 지원 (담당자 배정)
→ 2주간 전담 지원
→ 주 1회 현황 미팅
→ 교인 문의 즉시 응답
```

### 6.2 성도 온보딩 (B2C)

```
앱 다운로드 → 교회 코드 입력
                    ↓
            [3단계 온보딩]

1단계: 반가워요!
"안녕하세요! 말씀브릿지에 오신 걸 환영해요.
[교회명] 교인이시군요! 😊"
[시작하기]

2단계: 이렇게 사용해요
간단한 3개 슬라이드:
- "주일 설교를 5일로 나눠서"
- "매일 5-10분 묵상"
- "AI가 도와드려요"
[다음]

3단계: 알림 설정
"언제 알림을 받으시겠어요?"
- 아침: [09:00 ▼]
- 점심: [켜기/끄기]
- 저녁: [20:00 ▼]
[완료]
```

## 12. 부록

### 12.1 전도에서 성장까지: 사용자 여정

```
단계 1: 인지 (Awareness)
교회에서 앱 소개 → QR 코드 스캔 → 다운로드

단계 2: 활성화 (Activation)
첫 설교 확인 → 첫 묵상 완료 → AI 격려 수신

단계 3: 유지 (Retention)
주 3일+ 사용 → 완주 축하 → 습관 형성

단계 4: 성장 (Growth)
간증 나눔 → 기도 제목 공유 → 소그룹 활성화

단계 5: 추천 (Referral)
다른 교회 친구에게 추천 → 바이럴 성장
```

### 12.2 핵심 성과 지표 (KPI) 대시보드

```
회사 레벨 KPI:
- MRR (월간 반복 수익)
- 교회 수 (총 계약)
- MAU (월간 활성 사용자)
- Churn Rate (월간 이탈률 목표: 3% 미만)
- NPS (순 추천 지수 목표: 50 이상)

교회 레벨 KPI:
- 교인 앱 가입률
- 주간 참여율
- 완료율
- 간증 수

사용자 레벨 KPI:
- 연속 사용일
- 주간 완료율
- 묵상 메모 수
- 격려 메시지 반응률
```
