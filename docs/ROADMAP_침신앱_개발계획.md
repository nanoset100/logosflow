# 침신 말씀노트 앱 개발 로드맵
> 작성일: 2026-03-06 | 최종 업데이트: 2026-03-06

---

## 비전

> **침례교단 3500개 교회 교인들이 주일 설교를 개인 묵상과 구역 예배 교재로 연결하는 앱**

---

## 핵심 가치 제안 (2가지)

| 기능 | 대상 | 효과 |
|------|------|------|
| 설교 요약 | 개인 교인 | 주일 설교 복습, 매일 묵상 |
| 5일 묵상 | 구역장/소그룹 | 구역 예배 교재 자동 생성 (매주 2~3시간 절약) |

### 핵심 사용 시나리오

```
일요일   → 목사님 설교
월요일   → 앱 자동 생성: 설교 요약 + 5일 묵상
월~금    → 개인이 앱으로 매일 묵상
목~금    → 구역장이 앱 열어서 그대로 구역 예배 진행
(목요일) → 구역장이 카카오톡으로 교재 공유 → 앱 설치 유도 (바이럴)
```

---

## 배포 전략 (2-Tier)

### Tier 1: 대형교회 (신자 1000명 이상, 약 300개)
- 초기: 운영팀이 직접 Pastors.ai 활용 → 수동 입력
- 중기: 자체 AI 에이전트 자동 처리
- 방식: 유튜브 URL 입력 → 자동 요약/묵상 생성

### Tier 2: 소형교회 (신자 100명 이하, 약 2,500개 = 70%)
- 목사님 or 젊은 교인이 앱 내에서 직접 입력
- 유튜브 URL 붙여넣기 또는 설교 텍스트 붙여넣기
- AI가 자동으로 요약 + 묵상 생성 후 검토/발행

---

## 교단 배포 전략

- 침례교단 신학교 총장 친분 활용
- **교단 공식 앱**으로 인정받아 3500개 교회에 배포
- 교단 공식 인정 = 신뢰도 + 설치 장벽 제거

### Phase 1 - 관리자 CMS (설교 등록 시스템)
> 목표: 교회 관리자(목사/담당자)가 앱 내에서 직접 설교 등록

#### 1-1. 관리자 권한 시스템
- Firestore users/{uid} 에 role 필드 추가 (`admin` | `member`)
- 앱 내 관리자 탭 - role이 admin인 경우에만 표시
- 교회 코드별 관리자 분리 (A교회 관리자는 A교회만 관리)

#### 1-2. 설교 등록 화면 (3가지 입력 방식)
```
[설교 등록 화면]
├── 방법 1: 유튜브 URL 입력 → AI 자동 생성
├── 방법 2: 설교 텍스트 붙여넣기 → AI 자동 생성
└── 방법 3: 직접 입력 (요약/묵상 수동 작성)

공통 필드:
- 설교 제목
- 설교 날짜
- 설교자 이름
- 본문 성경 구절
```

#### 1-3. AI 생성 결과 검토 화면
```
[검토 화면]
├── 생성된 설교 요약 (편집 가능)
├── 생성된 5일 묵상 (편집 가능)
└── [발행] / [수정] / [취소] 버튼
```

### Phase 2 - AI 자동 생성 파이프라인
> 목표: 유튜브 URL → 요약 + 5일 묵상 완전 자동화

#### 2-1. 기술 스택
```
Flutter 앱 (관리자)
    │
    │ HTTPS 요청
    ▼
Firebase Cloud Functions (서버리스)
    │
    ├── YouTube Data API v3 (자막 추출)
    │   또는 yt-dlp (자막 없을 경우 Whisper STT)
    │
    └── Claude API (claude-sonnet-4-6)
            │
            ├── 설교 요약 생성 (3~5문단, 한국어)
            ├── 5일 묵상 생성 (성경구절 + 적용질문)
            └── 핵심 인용구 3개
                │
                ▼
          Firestore 저장
          churches/{code}/sermons/{id}
```

#### 2-2. Claude 프롬프트 설계 원칙
- 신학적 깊이 + 평신도 이해 가능한 언어
- 5일 묵상: 월(이해) → 화(적용) → 수(기도) → 목(실천) → 금(나눔) 구조
- 구역 예배 교재로 바로 사용 가능한 토론 질문 포함
- 한국 침례교 신학 맥락 반영

#### 2-3. 비용 예측
| 항목 | 비용 |
|------|------|
| Claude API (설교 1편 처리) | 약 $0.05~0.10 |
| Firebase Cloud Functions | 무료 티어 내 (초기) |
| YouTube API | 무료 (할당량 내) |
| 월 100개 교회 × 주 1회 | 약 $2~4/월 |

### Phase 3 - 구역 예배 교재 공유 기능
> 목표: 바이럴 확산 - 구역장이 교재 공유 → 앱 설치 유도

#### 3-1. 공유 기능
- "이번 주 교재 공유" 버튼
- 카카오톡 공유: 제목 + 본문 요약 + 앱 다운로드 링크
- PDF 내보내기: A4 1~2장 구역 예배 교재 형식
- 링크 공유: 앱 미설치자도 웹에서 열람 가능

#### 3-2. 구역 예배 교재 형식 (PDF/공유)
```
[이번 주 구역 예배 교재]
교회명 | 날짜 | 설교 제목
본문: 성경 구절

▣ 설교 요약
(3~4문단)

▣ 함께 나눔 (토론 질문 3개)
1. ...
2. ...
3. ...

▣ 이번 주 묵상 (5일)
월: ...
화: ...
수: ...
목: ...
금: ...

▣ 함께 기도
```

### Phase 4 - 교단 배포 및 확장
> 목표: 침례교단 3500개 교회 공식 배포

#### 4-1. 교단 공식 앱 인정 준비물
- 앱 소개 자료 (총장 프레젠테이션용)
- 파일럿 교회 5~10개 사용 후기
- 데이터 보안/개인정보 처리방침
- Play Store / App Store 정식 등록

#### 4-2. 교회 온보딩 프로세스
```
교단 공문 or 총회 발표
    │
교회 담당자 앱 설치 + 교회 코드 등록
    │
관리자 권한 부여 (총장 or 운영팀 승인)
    │
첫 설교 등록 (운영팀 지원)
    │
자체 운영 전환
```

#### 4-3. 수익 모델 (장기)
| 단계 | 모델 |
|------|------|
| 초기 | 무료 (교단 신뢰 확보) |
| 중기 | 교회 구독 (월 1~3만원/교회) |
| 장기 | 교단 계약 (단가 인하, 전체 패키지) |
| 부가 | AI 설교 생성 고급 기능 유료화 |

### Phase 5 - 자체 AI 에이전트 고도화
> 목표: Pastors.ai 수준을 넘는 한국 교회 특화 AI

#### 5-1. 추가 기능 (수요 확인 후 개발)
- 설교 시리즈 연결 묵상 (연속 설교 맥락 유지)
- 교인 연령별 맞춤 묵상 (청년부 / 장년부 / 노년부)
- 설교 기반 대표기도문 자동 생성
- 헌금 봉투 메시지 자동 생성
- 교회 주보 초안 자동 생성

#### 5-2. 데이터 자산화
- 3500개 교회 설교 데이터 축적
- 침례교 신학 특화 파인튜닝 가능성
- 향후 교단 신학교 교육 자료로 활용
